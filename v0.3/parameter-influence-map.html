PARAMETER INFLUENCE MAP

Inspector parameters and their system-wide effects
GROWTH RATE PARAMETERS (ManagerPlant1)
checkInterval
Default: 1.0 seconds
→ How often GrowthLoop evaluates all plants CRITICAL
→ Drought accumulation rate (timeInDrought += checkInterval)
→ Frequency of moisture removal from soil
⚠ Performance Impact: Lower values = more frequent checks = higher CPU usage. Values below 0.5s may cause frame drops.
growthIncrementLength
Default: 0.002 units
→ Base vertical growth speed HIGH
→ How tall plants can grow before death
→ Visual appearance (tall forest vs short grass)
↔ Interacts with: etiolationStrength (crowded plants grow 2x this value in height)
growthIncrementWidth
Default: 0.0001 units
→ Base thickness growth speed HIGH
→ Root radius expansion rate (wider plants = bigger root zones)
→ Competition priority (width weighted 2x in priority formula)
→ Number of neighbors detected (bigger roots = more neighbors)
⚠ Cascading Effect: Increasing this creates dominant plants that outcompete others (rich get richer).
rootDistanceMultiplier
Default: 1.0
→ How much root radius increases per width growth MEDIUM
→ Moisture access range (higher = can reach more distant tiles)
→ Neighbor detection range (higher = more crowding effects)
MOISTURE PARAMETERS (ManagerPlant1)
minMoisture / maxMoisture
Default: 0.3 / 0.7
→ Acceptable moisture range for plant growth CRITICAL
→ How many tiles are "usable" at any given time
→ Narrower range = more drought = more plant death
⚠ Ecosystem Stability: Range smaller than 0.3 units may cause mass die-offs during rain cycles.
optimalMoisture
Default: 0.5 (middle of range)
→ Target moisture for maximum growth rate HIGH
→ Center point for moisture quality calculation
→ Plants grow slower when moisture is far from this value
marginalMoistureGrowthRate
Default: 0.5 (half speed at edges)
→ Growth rate at edge of acceptable range MEDIUM
→ Penalty for non-optimal conditions
→ Lower values = plants more sensitive to moisture quality
moistureDecrement
Default: 0.01 per growth cycle
→ How much moisture each plant consumes HIGH
→ Rate of soil depletion
→ Balance with rain replenishment (must match rain rate)
↔ Must Balance With: Rain moistureIncrement and raindropCount. Too high = ecosystem collapse.
COMPETITION PARAMETERS (ManagerPlant1)
etiolationStrength
Default: 2.0 (double height, half width)
→ How extreme crowding effects are HIGH
→ Maximum length modifier (crowded plants grow this × taller)
→ Minimum width modifier (crowded plants grow 1/this × thinner)
→ Visual diversity (higher = more dramatic tall/thin vs short/wide)
⚠ Stability Risk: Values > 3.0 create extremely thin plants that may look unrealistic. Values < 1.5 reduce visual variety.
DEATH PARAMETERS (ManagerPlant1)
timeToDie
Default: 10 seconds
→ How long plants survive without moisture CRITICAL
→ Ecosystem resilience during rain gaps
→ Population stability (lower = more volatile)
↔ Scales With: ManagerTime.timeScale (10 seconds at 5x speed = 2 real seconds)
RAIN PARAMETERS (ManagerRain & ManagerSoil)
raindropCount (ManagerRain)
Default: 200 drops
→ Moisture replenishment rate HIGH
→ How quickly dry soil becomes wet
→ Must balance with plant moistureDecrement
moistureIncrement (ManagerSoil)
Default: 0.1 per drop impact
→ How much each raindrop adds to soil HIGH
→ Speed of moisture recovery after depletion
⚠ Balance Formula: (raindropCount × moistureIncrement) should roughly equal (plantCount × moistureDecrement × checkInterval)
PARAMETER TUNING PRINCIPLES: • Start with Ratios: Growth rate, moisture consumption, and rain replenishment must balance or ecosystem collapses.

• Test Extremes: Try 10x and 0.1x values to understand boundaries and failure modes.

• Watch Cascades: Changing growthIncrementWidth affects root radius → neighbor detection → etiolation → visual appearance.

• Time Scale Matters: Parameters with "time" units (checkInterval, timeToDie) are affected by ManagerTime.timeScale.

• Critical Trio: minMoisture/maxMoisture, timeToDie, and rain rate form the survival foundation. Tune these first.

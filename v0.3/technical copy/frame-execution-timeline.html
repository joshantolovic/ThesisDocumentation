UPDATE PHASE (Every Frame)
ManagerUI
Update()
• Check toggle key input
• Calculate FPS metrics
• Set isMouseOverUI flag
ManagerRain
Update()
• Check pause state
• Move all raindrops downward (speed × timeScale × deltaTime)
• Detect impacts → Fire OnDropImpact event
ManagerSoil
HandleDropImpact() [Event Callback]
• Convert world position to tile coordinates
• IncreaseMoisture() on impacted tile
COROUTINE PHASE (Timed Intervals)
ManagerPlant1
GrowthLoop() [Coroutine]
Frequency: checkInterval / timeScale (default: 1 second / timeScale)

Process:
• Wait for unpause
• For each plant:
  → Check CanGrow() (moisture availability)
  → If YES: Calculate growth modifiers → Apply growth → Remove moisture
  → If NO: Increment drought timer → Check for death threshold
DeathCycleCoroutine() [Coroutine]
Triggers: When plant timeInDrought ≥ timeToDie

Process:
• Random delay (1-5s / timeScale)
• Transition: Wilting → Decaying
• Another random delay
• Transition: Decaying → Dead
RENDER PHASE (Every Frame, After Update)
ManagerSoil
OnRenderObject()
• Draw all soil tiles with moisture-based colors
• Color lerp: dryColor → moistColor
ManagerRain
OnRenderObject()
• Draw all raindrop lines
• Position based on current Update() state
ManagerPlant1
OnRenderObject()
• Draw all plant lines (bottom → top)
• Color based on state (green/brown)
• Width/length from current growth values
GUI PHASE (Every Frame, After Render)
ManagerUI
OnGUI()
• Draw immediate-mode UI
• Handle button clicks
• Display stats and controls
KEY INSIGHTS: • No per-plant Update(): Plants are structs in a list, not GameObjects. GrowthLoop processes all plants periodically.

• Custom Time Scaling: Only affects GrowthLoop and DeathCycle (not Unity's physics/Update).

• Event-Driven Moisture: Rain impacts trigger immediate soil updates, not frame-by-frame checks.

• Render Independence: All rendering uses Shapes library in OnRenderObject() - no GameObject hierarchy.

MOISTURE COMPETITION STEP-BY-STEP

Priority-based system for contested soil moisture
COMPETITION PRIORITY FORMULA
Priority = seedVigor × (plantWidth × 2 + plantLength)
Higher priority = wins contested moisture
Width weighted 2x (represents root spread advantage)
STEP 1
Plant Needs Moisture
After a plant successfully grows, it must consume moisture from a nearby soil tile.

The plant calls RemoveMoistureAtNearest()
Goal: Find the closest suitable tile and remove moisture
Constraint: Must be within root radius
STEP 2
Calculate Own Priority
First, the plant calculates its own competition priority score.
myPriority = ManagerWorld.GetCompetitionPriority(
  seedVigor,
  plantWidth,
  plantLength
)
This score represents the plant's ability to "win" contested resources based on:
• Inherent strength (vigor)
• Size advantage (biomass = width × 2 + length)
STEP 3
Scan for Suitable Tiles
Loop through all soil tiles to find candidates:
For each tile:
  • Check if moisture is in acceptable range (minMoisture to maxMoisture)
  • Calculate distance from plant to tile
  • Check if distance ≤ root radius
  • Track the closest suitable tile
Root radius calculation:
rootRadius = defaultRootSpreadDistance + (rootDistanceMultiplier × widthGrowth)
STEP 4
Check for Competing Plants
For each candidate tile, check if other plants also have access to it:
For each other plant:
  • Skip if same plant (self)
  • Skip if not Healthy state
  • Calculate distance from other plant to tile
  • If distance ≤ other plant's root radius → COMPETITION!
If competition detected, compare priorities.
STEP 5
Priority Comparison
For each competing plant on the same tile:
otherPriority = ManagerWorld.GetCompetitionPriority(
  otherPlant.seedVigor,
  otherPlant.plantWidth,
  otherPlant.plantLength
)

if (otherPriority > myPriority):
  canTake = false // I lose, skip this tile
  break
Result: If any competitor has higher priority, this plant cannot use this tile.
STEP 6
Take Moisture or Move On
If plant has highest priority (or no competition):
• Mark this tile as a candidate
• Continue checking to find the closest suitable tile

After checking all tiles:
• Remove moisture from the closest tile where plant had highest priority
soilManager.DecreaseMoisture(x, y, moistureDecrement)
If no valid tiles found:
• Plant takes no moisture this cycle (will accumulate drought time)

Size advantage (width × 2 + length = 0.58) overcomes closer competitors.
KEY INSIGHTS: • Size Dominance: Larger plants almost always win contested moisture (width weighted 2x).

• Distance Doesn't Matter: Only used to determine IF a plant can reach a tile. Priority determines WHO gets it.

• Rich Get Richer: Big plants get more moisture → grow bigger → win more moisture. Creates dominance hierarchies.

• Multiple Checks: Each plant checks EVERY other healthy plant for EVERY tile. Computationally expensive but ensures fair competition.

• Zero-Sum Game: Only one plant can take moisture from a tile per cycle. Losers must find uncontested tiles or face drought.

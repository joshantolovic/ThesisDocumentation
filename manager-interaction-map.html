ManagerPlant1 → ManagerSoil: → Reads tileData[x,y] for moisture checks
Calls DecreaseMoisture() when plant consumes water
ManagerSoil → ManagerRain: → Subscribes to OnDropImpact event in OnEnable()
EVENT-DRIVEN COMMUNICATION
ManagerRain → ManagerSoil: ⚡ Fires OnDropImpact(Vector2) when raindrop hits ground
ManagerSoil handles event → IncreaseMoisture()
SINGLETON QUERIES
ManagerPlant1 → ManagerWorld: → Calls GetLocalDensity() for etiolation calculation
Calls GetCompetitionPriority() for moisture competition
ManagerWorld → ManagerPlant1: → Reads ManagerPlant1.Instance.plantSeeds list
Iterates plants for density/priority calculations
STATIC GLOBAL STATE
All Managers → ManagerTime: → Read ManagerTime.timeScale in coroutines
Used to scale wait times: checkInterval / timeScale
All Managers → ManagerUI: → Read ManagerUI.isPaused to halt simulation
Read ManagerUI.isMouseOverUI to block input
ManagerUI → All Managers: → Calls GenerateGridData(), GeneratePlantSeeds(), etc.
Direct method invocation for Restart/NewSeed buttons
ARCHITECTURAL PATTERNS: • Singleton Pattern: ManagerWorld and ManagerPlant1 use Instance for global access (competition queries).

• Event-Driven: Rain impacts don't poll - they fire events that Soil subscribes to.

• Static Flags: Pause and mouse state are static bools for easy access across systems.

• Direct References: Core dependencies (Plant→Soil, Soil→Rain) are inspector-assigned for clarity.

• Unidirectional Data: Soil doesn't know about Plants, Plants query Soil. Prevents circular dependencies.
